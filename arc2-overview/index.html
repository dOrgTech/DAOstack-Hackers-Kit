Arc 2.0¶

What's new on the latest version of DAOstack architecture?

Overview¶

  • Arc 2.0 significantly eases DAO deployment, plugin management, and general gas costs, all this while increasing security.
  • Its main objectives is to improve security, flexibility, user experience, and developer accessibility.

The changes¶

  1. None universal plugins
  2. All plugins are none universal which means a separate plugin instance per DAO.

  3. This reduces complexity at the contract level and increases security.

  4. All contracts are proxy upgradable contracts.

  5. Enable upgrades to DAO contracts implementations while maintaining addresses and storage.
  6. The DAO serves as the proxy admin of its own contracts.

  7. This increases security, UX, and enables upgradability.

  8. Arc.Hive

  9. ArcHive is an on-chain repository of all approved (Registered) DAOstack contract implementations.
  10. Contract/DAOs which have been registered / verified in the ArcHive will be automatically indexed and show up in Alchemy (higher layers of the stack)
  11. Plugins which would like to be added to the ArcHive need to be approved and verified by DAOstack. We aspire to have this managed by the community in the future.
  12. The ArcHive architecture increases security via a tool that validates contracts and implementation in a trust-minimized way.
  13. DAOFactory
  14. DAO deployment in a seamless process, in a single transacation
  15. DAO deployment cost is reduced by a factor of 10.
  16. Use ArcHive contracts to create a DAO
  17. DAO Factory DAOs will be indexed and will be shown in alchemy.
  18. DAO creation is made fast, Cheap and with far better UX.
  19. PluginFactory
  20. Simplified process of the “Plugin Manager” in updating / adding of plugins to the DAO.
  21. Plugin parameters are passed via the plugin add proposal.
  22. A new Plugin instance is created and deployed upon execution.
  23. GenesisProtocol parameters are set upon plugin Init, no need to pre-deploy governance parameters
  24. Easy and friendly UX
  25. Tx Cost
  26. Optimization was done to genesis protocol and plugins structure to reduce gas required for proposing/voting/staking.
  27. Subgraph auto indexing
  28. All Arc.Hive plugins/DAOs are automatically indexed by subgraph (no need to manually index).
  29. Misc
  30. NFT management plugin - send/mint/list
  31. Arc.react - enable easy custom react ui components on top of subgraph and arc.js
  32. JoinAndQuit and funding request
  33. Token Trade - Send and receive tokens from/to the organization.
  34. Key value on chain database
  35. Etherscan/blockscout contract verification upon DAO deployment.
  36. Full xDAI support with ERC20 token contract bridge.
